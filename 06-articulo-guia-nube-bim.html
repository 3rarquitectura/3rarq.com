Convertir nube de puntos a modelo BIM: la guía real

Convertir una nube de puntos en un modelo BIM ejecutable requiere cuatro etapas bien definidas. Esta es la guía completa del proceso.

1. Captura y Registro

1Generar la nube

El escáner láser genera millones de puntos con alta precisión. Software como Cyclone REGISTER 360 une las nubes de distintas estaciones, controla errores de unión y genera el dataset limpio y georreferenciado.

Resultado: Nube de puntos registrada, limpia de artefactos de captura, lista para la siguiente etapa.

2. Limpieza y Segmentación

2Preparar la nube para modelado

Se quita el ruido (reflexiones espurias, elementos móviles, interferencias), se segmenta por categorías (estructura, cerramientos, instalaciones, elementos singulares) y se reduce a la densidad necesaria para el modelado.

Por qué es importante: Reduce volumen de datos, agiliza el trabajo de modelado y evita que el modelador pierda tiempo interpretando ruido.

3. Modelado BIM

3Construir el modelo inteligente

En Autodesk Revit u otro software BIM, el modelador interpreta la nube y construye el modelo con el nivel de detalle acordado (LOD 200, 300 o 400). Los elementos se crean como familias paramétricas clasificadas por disciplina.

Decisiones clave en esta etapa:

  • ¿Qué LOD? (Define precisión geométrica y nivel de información)
  • ¿Qué disciplinas? (Arquitectura, estructura, MEP)
  • ¿Cómo se resuelven las ambigüedades de la nube?

4. Exportación y Documentación

4Entregar en formato ejecutable

El modelo se exporta en IFC (Industry Foundation Classes) para interoperabilidad. Se generan los planos de relevamiento, cuantificaciones y los entregables específicos que el proyecto requiere.

Entregables típicos: Archivos nativos Revit (.RVT), exportación IFC, planos PDF, tablas de información, documentación As-Built.

Elementos clave para el éxito

  • Definir el LOD antes de empezar. LOD 200, 300 y 400 tienen implicancias muy distintas en tiempo, costo y precisión.
  • Acordar alcance disciplinar. ¿Solo arquitectura, o también estructura e instalaciones?
  • Establecer protocolos de entrega. Compatibles con el software y el workflow del cliente.
  • Documentar ambigüedades. Zonas de la nube donde hay datos faltantes o reflexiones: cómo se resolvieron en el modelo.

Cronograma típico

Edificio de 500 m² con LOD 200 (arquitectura): 40–80 horas de modelado (1–2 semanas).

LOD 300 con estructura e instalaciones: 80–160 horas de modelado (2–4 semanas).

Los tiempos varían según complejidad de la geometría, densidad de detalles y claridad de la nube.

Preguntas frecuentes

¿Es posible hacer Scan-to-BIM sin visitar el edificio?

Sí, si la nube está bien capturada y registrada. El modelador trabaja directamente desde la nube; las ambigüedades se documentan y resuelven con el cliente o con una visita complementaria acotada.

¿Cuál es la diferencia entre LOD 200, 300 y 400?

LOD 200 es geometría aproximada (suficiente para estimaciones). LOD 300 es geometría precisa (para coordinación). LOD 400 es detalles constructivos (para fabricación o presupuestos exactos).

¿Qué pasa si la nube tiene zonas ocultas?

Se documentan explícitamente. El modelo muestra qué se modeló con precisión y qué se interpoló o simplificó por falta de datos.
